# issue イントネーション、上下の編集範囲max minをoverするとscrollが発生してUXが悪くなる #62
[issues #62](https://github.com/cat2151/voicevox-playground/issues/62)

- mouse操作中は、scrollは禁止とする。userが混乱する
- maxまでマウスを動かしたらmaxに張り付くようにせよ、minも同様
- 上下の幅は、生成時の初期化のあとは、2x と 0.5x でのみ変化するようにせよ
- あわせて、キーボード操作ボタンの行に、「リセット」ボタンを追加し、レスポンスで得たイントネーションにリセットできるようにせよ
- レイアウト改善。2xボタンなどの横幅がムダに広すぎる。文字が表示できる最小限の横幅まで縮めること。そしてそのぶんイントネーション編集グラフ領域の横幅を広げること。
- レイアウト編集時は、spaceとenterを押すと明示的にcacheを再生すること
- キーボード上下の変化量が大きすぎる。最初にレスポンスで得たイントネーションminとmaxの間を10分割した1段階、で変化させること
    - SHIFT 上下でその1/2の変化量になること
    - CTRL 上はminとmaxを前述の1段階ぶん広げること。
    - CTRL下は同様に1段階ぶん狭くすること。
        - ただし現在のイントネーションのモーラpitchの最小値と最大値がおさまる範囲までしか狭くできないようにする。
        - 「これによって現在イントネーションのモーラpitchが破壊されて変化する」ことを禁止する。それはUXが悪い。
